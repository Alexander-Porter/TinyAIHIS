TinyHIS 医院信息系统设计说明书
1. 引言 (Introduction)
1.1 项目背景
TinyHIS 旨在为中小型医疗机构提供一套轻量级、智能化、全流程的信息管理系统。系统采用 B/S 架构，覆盖患者就诊、医生工作站、医技协同及行政管理等核心业务场景，并引入 AI+RAG（检索增强生成）技术辅助分诊，提升服务效率。
1.2 技术选型 (Tech Stack)
后端核心: Java 21, Spring Boot
持久层: MyBatis-Plus, MySQL 8.0
缓存与消息: Redis (用于会话管理、队列缓冲)
实时通信: WebSocket
前端架构: Vue 3 + TypeScript
AI/RAG支持: Spring AI，配合关键词检索（不需要向量检索），对接硅基流动LLM。

2.2 核心业务流程
就诊流程: 注册 -> AI分诊 -> 挂号(缴费) -> 到院签到 -> 进入排队队列 -> 叫号 -> 医生看诊 -> 开单 -> 缴费 -> 检查->医生查看检查结果->开药->取药-> 离院。


你需要仔细理清以下实体的关系：
药品、药品模板、用户、病人、医生、主任医生、检验科医生、药房医生、检验项目、检验结果、病例本、就诊记录、医嘱、科室、排班、就诊室。

UI设计：
1.图标与符号：不要使用Emoii，全部替换为专业的图标(lcons)。
2.布局与间距：重新调整内边距 (Padding)，让组件之间的距离完美平衡——既不要挤在一起，也不要过于分散而浪费空间。
3.核心风格：网站的目标是看起来流畅、高端且极简，就像一家位于瑞士的高级水疗中心(Swiss Spa)那样纯净。
4.价值感：设计质感必须让人觉得物有所值，要达到“专业人士愿意每月为此支付数千美元”的水准。审美标准要高到能让史蒂夫·乔布斯(Steve Jobs)看了都会会心一笑。
5.色彩管理：避免使用不必要的颜色。请选择一套协调且克制的调色板，并严格遵守它。
6.响应式设计：确保网站在桌面端和移动端的展示都同样优雅、灵敏。”
